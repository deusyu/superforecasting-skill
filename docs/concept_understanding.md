# Superforecasting 概念理解

> 基于本次对话与用户提供的课程文本整理。  
> 目标：为 Superforecasting Skill 提供概念基础、术语解释和工作流理解。

---

## 1. Superforecasting 是什么？

**Superforecasting，中文可译为“超级预测”或“超预测”，是一种把未来判断变成可量化、可检验、可更新、可复盘的概率预测方法。**

它的核心不是“神奇地知道未来”，而是建立一个训练判断力的闭环：

```text
把模糊判断变成概率
→ 把概率绑定到明确事件和截止日期
→ 随着新信息更新概率
→ 到期后结算
→ 用记分结果反过来校准自己
```

最简公式：

```text
Superforecasting = 概率化 + 可检验
```

---

## 2. 它不是什么？

Superforecasting 不是：

1. **不是占卜。**
   - 它不声称能看见未来。

2. **不是专家表态。**
   - “我认为会发生”不是超级预测。
   - 超级预测要求给出概率、截止日期、判定标准。

3. **不是情绪安慰。**
   - 它不负责让人感觉舒服，而是让判断变得可校准。

4. **不是一次性建议。**
   - 预测是动态的，会随着证据更新。

5. **不是纯模型崇拜。**
   - 它可以使用模型，但并不依赖复杂模型。
   - 很多时候，参考类 + 贝叶斯更新已经能显著改善判断。

---

## 3. 为什么要概率化？

普通判断常常是这样的：

```text
美国很可能会对伊朗越来越强硬。
这个产品应该会火。
我搬去广州应该会幸福。
客户大概率不会毁约。
```

这些表达的问题是：

```text
1. 没有明确概率
2. 没有截止日期
3. 没有判定标准
4. 事后很难结算
5. 错了也容易解释成“我不是那个意思”
```

超级预测要求把它们改成：

```text
到某年某月某日之前，美国是否会对伊朗目标实施军事打击？概率是多少？
上线后 30 天内，产品次日留存是否超过 35%？概率是多少？
搬去广州 6 个月后，生活满意度是否达到 7/10？概率是多少？
明天 18:00 前，客户是否明确取消合同？概率是多少？
```

概率化的价值是：

```text
没有概率，就没有校准。
没有校准，就没有学习。
没有结算，就没有进步。
```

---

## 4. 可检验是什么意思？

一个预测要可检验，至少需要四个要素：

```text
1. 明确事件：预测什么？
2. 截止日期：到什么时候为止？
3. 判定标准：什么算发生，什么算没发生？
4. 数据来源：根据什么结算？
```

例子：

```text
差：这个产品会不会受欢迎？
好：到 2026-08-01，新用户次日留存率是否超过 35%，以产品后台数据为准？
```

```text
差：我去广州会幸福吗？
好：到 2026-12-31，我在广州连续居住至少 4 个月后，主观生活满意度是否达到 7/10 以上？
```

---

## 5. 费米化：把云状问题拆成小问题

费米化来自费米估算的思想：面对大而模糊的问题，不是直接猜答案，而是先把问题拆成若干可操作的小问题。

课程文本中把模糊问题称为“云状问题”：边界模糊、无法直接判断对错。

例子：

```text
中美会不会脱钩？
```

这是云状问题，因为“脱钩”本身不清楚。费米化后可以变成：

```text
未来六个月内，某个关税法案通过的概率是多少？
未来一年内，某项核心技术贸易额下降超过 10% 的概率是多少？
未来一年内，双方是否新增关键技术出口限制？
```

人生决策例子：

```text
我去广州会幸福吗？
```

费米化后：

```text
1. 3 个月内，能否找到月薪不低于当前 80% 的工作？
2. 1 个月内，能否找到预算内且通勤可接受的住处？
3. 6 个月内，能否建立稳定社交圈？
4. 3 个月后，是否适应广州气候和生活节奏？
5. 6 个月后，生活满意度是否达到 7/10？
```

费米化的意义是：

```text
不确定性一旦被拆解和命名，就从一种情绪变成了工程问题。
```

## 6. 外部视角：先找参考类

超级预测不鼓励一上来讲个人故事，而是先问：

```text
类似事情过去通常怎样？
```

这叫外部视角，也叫参考类思维。

例子：

```text
问题：我能在广州 3 个月内找到月薪不低于当前 80% 的工作吗？
```

先不要急着说：

```text
我很努力。
我能力不错。
我有信心。
```

先找参考类：

```text
过去从北京迁往广州的同职业求职者，3 个月内找到合适工作的比例是多少？
同年龄段、同收入阶段的人，跨城求职成功率是多少？
有内推资源的人，求职周期一般多长？
```

参考类可以分三层：

```yaml
broad: 所有跨城市求职者
medium: 从北京迁往广州的同职业求职者
narrow: 同行业、同年限、同收入阶段、且有内推资源的人
```

通常不要只用最窄参考类，因为样本可能太少；也不要只用最宽参考类，因为可能不够贴近。比较稳妥的是选择一个中等参考类，再用宽窄参考类做对照。

## 7. 内部视角：再看这个事件的特殊性

有了基础概率之后，再加入当前事件的具体信息。

例子：

```text
基础概率：类似求职者 3 个月内找到合适工作的概率是 40%。
```

内部视角上调因素：

```text
1. 有前同事内推
2. 有稀缺技能或核心专利
3. 目标行业正在扩招
```

内部视角下调因素：

```text
1. 广州同类岗位少
2. 当前市场招聘偏冷
3. 对薪资下限要求较高
```

于是可以从 40% 调整到 60%-68%，取当前概率 65%。

关键原则：

```text
先外部视角打底，再内部视角微调。
不要一上来就被自己的故事带走。
```

## 8. 贝叶斯更新：预测是动态的

超级预测不是一次性报数，而是不断更新。

基础概率相当于先验。随着新证据出现，要调整概率。

例子：

```text
初始概率：43%
出现利好新闻：上调到 46%
出现利空消息：下调到 41%
```

这体现的是稳健更新：

```text
不听而不闻，也不听风就是雨。
```

在实际 skill 中，不一定每次都使用严格公式。可以采用证据强度分档：

```yaml
strong_upward: +10% to +20%
moderate_upward: +5% to +10%
weak_upward: +2% to +5%
neutral: 0%
weak_downward: -2% to -5%
moderate_downward: -5% to -10%
strong_downward: -10% to -20%
```

但要避免机械调参。一个证据对概率的影响取决于：

```text
1. 证据是否可靠
2. 证据是否新鲜
3. 证据是否独立
4. 证据是否真正影响结局
5. 当前概率已经有多接近 0 或 1
```

## 9. 校准度 Calibration

校准度衡量的是：你给出的概率是否长期匹配现实频率。

例如：

```text
你曾经做过 100 个 70% 的预测。
如果其中大约 70 个真的发生了，你在 70% 档是校准的。
如果只发生了 50 个，你过度自信。
如果发生了 90 个，你过度保守。
```

校准度不是看单题押没押中，而是看长期表现。

这点非常关键：

```text
一个 70% 的预测失败了，并不必然说明预测错得离谱。
因为 70% 本来就意味着还有 30% 的可能不发生。
```

真正要看的是一组预测的长期表现。

---

## 10. 区分度 Resolution

区分度衡量的是：预测者有没有能力把高概率事件和低概率事件区分开。

如果一个人所有事情都报 50%，他可能看起来“稳健”，但没有信息量。

好的预测者应该敢于在证据充分时给出：

```text
15%、25%、70%、85%
```

而不是永远停在：

```text
45%-55%
```

不过区分度不能以牺牲校准度为代价。不是越极端越好，而是在该极端时极端，在不确定时承认不确定。

## 11. Brier Score

Brier Score 是评估概率预测的常用记分方法之一。

对于二元预测：

```text
Brier Score = (p - outcome)^2
```

其中：

```text
p = 预测概率
outcome = 实际结果，发生为 1，未发生为 0
```

例子 1：预测发生概率 70%，结果发生。

```text
Brier = (0.70 - 1)^2 = 0.09
```

例子 2：预测发生概率 70%，结果未发生。

```text
Brier = (0.70 - 0)^2 = 0.49
```

分数越低越好。

Brier Score 的价值是：

```text
1. 惩罚过度自信
2. 鼓励概率校准
3. 让预测变成可训练技能
4. 让长期复盘成为可能
```

## 12. 预测和决策必须分开

预测问的是：

```text
这件事发生的概率是多少？
```

决策问的是：

```text
在这个概率下，我要不要行动？
```

两者不同。

例子：

```text
预测：搬去广州后 6 个月生活满意度 >= 7/10 的概率是 62%。
```

这并不自动等于：

```text
应该马上搬去广州。
```

还要看：

```text
1. 行动成本
2. 可逆性
3. 失败损失
4. 备选方案
5. 后悔成本
6. 风险承受能力
```

所以 skill 应输出行动阈值，而不是直接替用户做决定：

```yaml
decision_threshold:
  act_if_above: 0.70
  test_if_between: [0.55, 0.70]
  pause_if_below: 0.55
```

## 13. 超级预测者的典型能力

超级预测者通常有这些能力：

```text
1. 概率表达能力
2. 费米化拆解能力
3. 参考类选择能力
4. 贝叶斯更新能力
5. 证据权重判断能力
6. 校准意识
7. 复盘习惯
8. 智识谦逊
9. 反身份化能力
10. 对模糊词的警惕
```

其中最适合做成 skill 的能力是：

```text
1. 把云状问题拆成子问题
2. 把子问题改写为可结算问题
3. 强制给出概率
4. 明确更新触发器
5. 到期后评分复盘
```

## 14. AI 在超级预测中的位置

AI 不应该被设计成“替人预言未来”的机器，而应该被设计成预测助理。

AI 擅长：

```text
1. 帮用户发现问题太模糊
2. 生成费米化拆解
3. 提出可能的参考类
4. 整理正反证据
5. 生成更新触发器
6. 维护预测卡
7. 计算 Brier Score
8. 总结复盘教训
```

人仍然重要，因为人负责：

```text
1. 界定真正关心的问题
2. 判断结算标准是否合理
3. 选择最合适的参考类
4. 判断异常证据是否重要
5. 设定行动阈值
6. 承担决策责任
```

理想关系不是：

```text
AI 替人预测
```

而是：

```text
AI 帮人把预测流程工程化，人保留最终判断权。
```

## 15. 为什么说它是“社会科学的工程化”？

传统评论常常追求解释：

```text
事情发生后，讲一个合理叙事。
```

超级预测追求的是：

```text
事情发生前，给一个可结算概率。
```

它把多个因素纳入一个工作流：

```text
问题定义
→ 变量拆解
→ 参考类
→ 概率估计
→ 新证据更新
→ 到期结算
→ 评分复盘
```

这更像工程，而不只是评论。

它的关键不是模型多复杂，而是流程足够严格：

```text
1. 不允许模糊表态
2. 不允许无限期预测
3. 不允许事后改口
4. 不允许只讲叙事不记分
5. 不允许只看单次成败
```

## 16. Superforecasting Skill 的概念抓手

如果要把超级预测做成 skill，最重要的不是让模型“更聪明”，而是让模型始终执行以下约束：

```text
1. 模糊问题必须改写
2. 云状问题必须费米化
3. 每个预测必须有截止日期
4. 每个预测必须有判定标准
5. 先外部视角，后内部视角
6. 概率必须可更新
7. 预测必须能结算
8. 结果必须能记分
9. 复盘关注校准，而不是面子
10. 决策和预测分开
```

这就是 skill 的概念内核。

## 17. 常见输入与转化方式

### 17.1 人生焦虑

```text
输入：我去广州会幸福吗？
转化：到某日期，搬去广州后生活满意度是否达到 7/10？
进一步拆解：工作、居住、社交、气候、财务。
```

### 17.2 产品判断

```text
输入：用户会喜欢这个功能吗？
转化：上线后 30 天内，该功能周活使用率是否超过 20%？
进一步拆解：激活率、留存率、复用率、付费转化、用户访谈。
```

### 17.3 商业风险

```text
输入：客户会不会毁约？
转化：到明天 18:00 前，客户是否明确取消当前合同？
进一步拆解：预算、付款流程、合同条款、沟通频率、替代供应商。
```

### 17.4 学习目标

```text
输入：我下次模考数学能上 120 吗？
转化：到下次模考日期，数学分数是否 >= 120？
进一步拆解：最近三次成绩、错题类型、复习时长、考试难度。
```

## 18. 术语表

| 术语 | 含义 |
|---|---|
| 概率化 | 用数字概率表达预测，而不是用模糊词 |
| 可检验 | 有明确截止日期、判定标准、数据来源 |
| 云状问题 | 边界模糊、无法直接判定对错的问题 |
| 费米化 | 把大问题拆成可操作的小问题 |
| 参考类 | 与当前事件相似的一组历史事件 |
| 外部视角 | 先看类似事情通常怎样 |
| 内部视角 | 再看当前事件的特殊性 |
| 先验 | 初始概率，通常来自参考类 |
| 贝叶斯更新 | 根据新证据调整概率 |
| 校准度 | 预测概率与长期真实频率是否匹配 |
| 区分度 | 是否能区分高概率和低概率事件 |
| Brier Score | 衡量概率预测误差的平方损失 |
| 行动阈值 | 到达什么概率才采取某行动 |
| 预测卡 | 一次预测的结构化输出 |
| 判断账本 | 保存预测、更新、结果和复盘的记录系统 |

---

## 19. 最短总结

Superforecasting 的精神可以压缩成三句话：

```text
不要说“会不会”，要说“概率是多少”。
不要说“未来会怎样”，要说“到哪天、什么算发生”。
不要把预测当观点，要把预测当账本。
```

最终目标不是预测每件事都对，而是让判断力在一次次结算中变得越来越校准。
